# Codex 自动视频剪辑工作流 Skill

这个 Skill 用来让 Codex 把已经录好的原始口播视频、配套文案或主题简报，剪成可发布的短视频解释类成片。

## 核心输入

默认把“口播视频”作为主输入。用户应先录好口播，再把视频放入输入文件夹；如果有正式文案，放同名 `.txt`、`.md`、`.docx` 或 `.srt` 到同一文件夹。

推荐输入：

```text
input/
  视频名.mp4
  视频名.txt
bgm/
  背景音乐.mp3
output/
```

如果没有口播视频，只有文案或主题简报，可以辅助规划素材和生成视频方案，但不能默认承诺完整自动出片效果等同于口播剪辑。

## 适用范围

最适合：

- 真人口播、访谈口播、知识讲解口播、长视频口播、短视频口播；
- 财经、美股、商业、AI、技术、创业、知识分享、课程培训、企业号、老板 IP、直播切片类解释视频；
- 有正式文案，需要按文案校准字幕和素材的口播；
- 需要删除停顿、重说、错话、重复观点的原始素材；
- 需要加字幕、BGM、轻音效、B-roll、全屏 cutaway 和动态信息贴片的短视频。

也适合：

- 多段口播合并；
- 录屏教程 + 口播讲解；
- 产品介绍、软件演示、网页/GitHub/数据页面讲解；
- 4:3、9:16、16:9 等不同平台规格。

不作为默认流程：

- 没有清晰人声的纯音乐混剪；
- 复杂剧情短片、广告片、电影级后期；
- 只有素材、没有口播或文案的泛视频生成。

## 核心原则

规则是累积的。用户新增的反馈只能叠加到当前规则栈里，不能把旧规则忘掉；除非用户明确取消某条旧规则。

执行顺序固定：

```text
1. 新电脑或首次使用时，先做环境检查和首次使用向导。
2. 让用户选择目标规格：4:3、竖屏 9:16、横屏 16:9，或保持源比例。
3. 确认源视频、文案、目标比例、输入/输出/BGM/素材目录。
4. 如果是多段视频，先做源文件预检，确认 1/2、上/下、part1/part2 没混错。
5. 先清理口播：停顿、重复、错话、重说、半句话、低信息量片段。
6. 先做顺畅的口播时间线，再加素材。
7. 默认口播清理后做 1.2 倍语速，保持音高不变。
8. 字幕和素材时间点必须对齐最终清理后的音频，不能沿用原始 ASR 时间。
9. 按最终文案或清理稿拆成素材需求点。
10. 素材必须同时通过“句子级匹配”和“好看”两道门。
11. 混合使用真人、真人加动态信息贴片、大素材、全屏资料卡、真实/视频化 cutaway。
12. 正式成片默认加字幕、轻快低音量 BGM、轻量且不重复的音效。
13. 交付前必须做规则审计、技术 QA、抽帧/事件 QA。
```

不能用素材、音乐、音效去掩盖口播时间线的问题。

## 安装和首次使用体验

新用户第一次安装后，先让流程变清楚，再开始剪片：

```text
1. 检查环境：FFmpeg、Python、转写/对齐工具、输入/输出/BGM/报告目录。
2. 选择规格：4:3、9:16、16:9 或保持源比例。
3. 确认输入：口播视频、同名正式文案、BGM、素材目录。
4. 说明缺什么：不要静默安装工具或下载大文件，先请求用户同意。
```

推荐引导语：

```text
使用 $auto-video-editing-workflow，先检查环境，创建输入/输出/BGM/报告目录，并让我选择成片规格。
```

## 质量闭环

正式成片必须同时通过四条质量线，不能只做好其中一部分：

```text
口播清理：停顿、重复、错话、重说、废话先处理干净。
素材匹配：B-roll、cutaway、贴片必须贴当前句子，同时好看、不重复、不弱相关。
字幕/BGM/音效：字幕按最终音频重新对齐；BGM 轻快不压人声；音效轻、少、变化。
QA 审计：记录素材来源、授权风险、规则审计、抽帧/事件图、解码检查。
```

如果新增风格导致任一质量线消失，例如只有全屏素材没有动态贴片、只有贴片没有真实素材、素材漂亮但不匹配、字幕和声音漂移、BGM 压人声、没有 QA 报告，都必须降级为测试版并继续返工。

## 必读引用

按任务读取，不要一次性塞满上下文：

- 真实剪辑任务：读 `references/workflow.md`。
- 视觉、素材、贴片、BGM、音效：读 `references/style-rules.md`。
- 最新底层逻辑、比例规则、素材站规则、字幕对齐、多段源视频预检：读 `references/bottom-logic.md`。
- 财经、美股、商业、技术解释视频：读 `references/finance-explainer.md`。
- 本机路径、安装、检查、交付：读 `references/local-setup.md`。
- 正式导出前：读 `references/non-regression-checklist.md`。

## 默认行为

- 当前默认成片比例：首次使用必须提醒用户选择；如果用户不选，才使用 4:3，1440x1080，30fps，MP4。
- 4:3 适合用户当前的小林说式横板讲解画面：1440x1080。
- 用户明确要求竖屏时，用 9:16，1080x1920。
- 用户明确要求横屏时，用 16:9，1920x1080。
- 用户想保留原视频比例时，用 source，但仍要做清理、字幕、素材和 QA。
- 默认口播速度：清理后 1.2 倍，保持音高。
- 默认正式成片：内嵌字幕、低音量轻快 BGM、轻音效。
- 默认素材节奏：真人约 45%-55%，素材主导约 45%-55%。
- 默认 2 分钟左右财经/商业/技术解释视频：规划 5-7 个真实或视频化全屏 cutaway，通常 6 个。
- 默认真人段落：需要保留中大号动态信息贴片，但不能贴脸、不能小贴纸化。

## 源视频预检

如果用户说“最新视频”，按输入目录修改时间排序，但不能只看时间。需要检查文件名、时长、首尾帧和开头/结尾转写片段。

如果是多段视频，例如 `AI美女1` + `AI美女2`、`part1` + `part2`，必须确认：

```text
文件名
文件大小
时长
首帧/尾帧
开头转写
结尾转写
主题是否连续
```

如果发现其中一段是别的主题，必须停止并报告，不能硬混剪。

## 口播清理

先做口播文本审查，再剪视频。

必须删除：

- 长停顿、等待、无讲话段；
- 半句话、犹豫词、口误残片；
- 错话后又重说的前一版；
- 同一句或同一个观点附近重复；
- 股票、ETF、公司列表重复念；
- “市场，市场”“纳指期货，纳指期货”这类开头词重复；
- 没有新信息的总结性废话。

优先保留：

- 后面更完整的一版；
- 数字、对象、结论更清楚的一版；
- 保留“第一、第二、第三、最后”等结构标记；
- 能让逻辑更顺的过渡句。

## 文案和字幕

如果同目录有同名 `.txt`、`.md`、`.docx`、`.srt`，或明显叫“文案”“稿子”“script”“manuscript”的文件，优先作为字幕和文案校准依据。

规则：

- 文案是内容真值，ASR 主要用于找时间点。
- 不要把原始 ASR 直接烧成字幕。
- 口播清理、剪切、1.2 倍变速后，要重新对齐字幕。
- 素材事件要绑定到对齐后的句子时间点。
- 字幕不能明显早于或滞后于声音。
- 字幕必须短、可读，不遮脸、不遮嘴、不遮关键素材文字。

## 素材选择

素材必须同时满足：

```text
句子级匹配
视觉好看
不重复
动态化或真实视频化
来源/授权可记录
```

素材优先级：

```text
S 级：用户自有实拍、自己录屏、官方公开视频、产品/网页/软件真实录屏。
A 级：正规授权 stock video、官方页面、公开媒体、可确认许可证的公开视频/图片。
B 级：AI 短视频、电影感生成场景、高质量照片做强运动。
C 级：静态卡片、PPT、假 UI、纯文字卡，只能少量兜底。
```

素材站/API 只生成候选池，不能直接进时间线。每个外部素材都要记录来源、作者、许可证、下载时间、搜索词、匹配句子、匹配分、美感分、商用风险和人工审核标记。

## 视觉风格

不要把成片做成 PPT 轮播。目标是成熟短视频解释类风格：

- 真实视频；
- 真实场景；
- 电影感照片/动图；
- 官方页面和产品录屏；
- 数据图动态绘制；
- 大尺寸资料镜头；
- 真人段落加中大号信息贴片；
- 少量必要的观点卡或结构卡。

禁止：

- 全片都是 PPT 卡片；
- 小标签、小贴纸贴在脸旁边；
- 同一个素材反复出现；
- 同一个模板、颜色、动画反复出现；
- 泛办公室、泛 AI、泛财经素材硬塞；
- 军人、战争、警察、武器、监狱、暴力冲突素材乱入；
- 真实视频上额外加进度线/扫描线来假装高级。

## 动态信息贴片

用户需要这类贴片，但要做成“内容信息层”，不是小装饰。

可用类型：

- 关键数字卡；
- 风险提示卡；
- 对比卡；
- 步骤卡；
- 结论卡；
- 小流程；
- 时间线；
- 1-3 个素材缩略图组合。

规则：

- 中大号、清晰、可读；
- 不遮眼睛、嘴、脸、手和字幕；
- 动画要变化，不要每次同一位置同一声音；
- 文字短，贴当前句子；
- 真人可见段落中约 60% 可以有贴片，但不能把视频变成 PPT。

## QA 交付

正式导出前至少检查：

```text
口播清理是否通过
1.2 倍语速是否正确
字幕是否对齐最终音频
BGM 是否低于人声且情绪轻快
音效是否轻、少、变化
素材是否句子级匹配
素材是否好看
素材是否不重复
是否有视觉相似重复
是否有真实或视频化 cutaway
是否仍像 PPT
是否有贴脸小贴纸
是否有素材来源表
是否有抽帧图/事件图
ffmpeg/ffprobe 解码是否通过
```

QA 报告必须明确给出四条质量线结论：

```text
speech_cleanup_pass
material_match_pass
caption_bgm_sfx_pass
final_qa_audit_pass
```

如果 QA 不通过，必须返工后再交付。

## 工具脚本

运行 `scripts/check-video-workflow.ps1` 检查本机路径、工具和常见脚本。

新电脑缺少工具时，先说明缺什么并请求用户同意，再运行 `scripts/setup-video-workflow.ps1`。不要静默下载安装 FFmpeg、Python 包或大模型。

首次配置示例：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\setup-video-workflow.ps1 -TargetProfile ask
```

如果用户已经明确平台规格，可以直接传：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\setup-video-workflow.ps1 -TargetProfile 4x3
powershell -ExecutionPolicy Bypass -File scripts\setup-video-workflow.ps1 -TargetProfile vertical
powershell -ExecutionPolicy Bypass -File scripts\setup-video-workflow.ps1 -TargetProfile horizontal
```
